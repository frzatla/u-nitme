# U-NIT ME — CLAUDE.md

## Project Overview

**U-NIT ME** is a Next.js web app that helps Monash University students plan their courses using a Python scheduling algorithm. Built for a Unihack hackathon. Users sign in, fill in their degree details, and get a semester-by-semester course plan generated by `algo1.py`.

## Full User Flow

1. User signs in via **Clerk**
2. Dashboard shows existing saved plans with stats
3. User clicks "New Plan" → goes to `/profile`
4. User fills in: Plan Name, University, Course (dropdown), Area of Study (filtered by course), Offering, Start/End year
5. On submit → server action in `profile/page.tsx`:
   - Runs `algo1.py` via `spawnPython()` (tries `py` → `python` → `python3` on Windows; `python3` → `python` → `py` on Unix)
   - algo1.py writes `schedule_<uuid>.json` to `src/algo/`
   - Server reads and parses the file, then deletes it
   - Enriches each unit with a `category` field (`Specialisation` if in AOS `all_units`, `Elective` if code=`ELECTIVE`, else `Core`)
   - Saves plan with `schedule` and `saved: false` to Supabase
   - Redirects to `/course-plan?planId=<uuid>`
6. Course plan page loads plan by `planId` query param, renders real schedule via `CoursePlanner`
7. User clicks a unit card → `UnitReviewModal` side panel slides in from the right, fetches `/api/units/[code]/reviews`, shows r/Monash Reddit threads with score + comment count as clickable links. `ELECTIVE` placeholder cards are not clickable.
8. User clicks **Save Plan** → server action marks `saved: true` → redirect to dashboard

## Key Data Models

```typescript
// lib/types.ts
type Plan = {
  id: string;
  planName: string;
  courses: string;       // course_code from final_courses.json (e.g. "C2001")
  university: string;
  areaOfStudy: string;   // AOS code from final_aos.json (e.g. "ALGSFTWR01")
  semesterOffering: string;
  yearStart: number | string;
  yearEnd: number | string;
  schedule?: Schedule;   // populated after algo1.py runs
  saved?: boolean;       // true only after user clicks Save
};

type UnitCategory = "Core" | "Major" | "Minor" | "Elective" | "Specialisation";

type ScheduledUnit = {
  code: string;
  title: string;
  credit_points: number;
  level: number | null;
  chain_length: number | null;
  extended: boolean | null;
  category: UnitCategory;   // enriched server-side after algo run
};

type ScheduledSemester = {
  semester: string;       // "Year 1, Semester 1"
  period: string | null;  // "S1" | "S2" | null (null = unscheduled bucket)
  semester_index: number;
  units: ScheduledUnit[];
  fixed_cp: number;
  total_cp: number;
  cumulative_cp: number;
};

type Schedule = {
  course_code: string;
  course_title: string;
  specialisation: string | null;
  major: string | null;
  minor: string | null;
  campus: string;
  summary: { total_units, total_cp, scheduled_cp, semesters };
  schedule: ScheduledSemester[];
};
```

## Student Details Form

**`StudentDetailsForm.tsx`** (server component):
- Reads `final_courses.json` and `final_aos.json` via `fs.readFileSync`
- Filters AOS to only entries with `total_credit_points > 0`
- Builds `courseToAos` map: a group contributes AOS codes only if `credit_points > 0` AND every unit in the group is a valid AOS code (excludes mixed FIT/MAT groups like "Discipline elective studies")
- Passes `courses`, `aosList`, `courseToAos` as props to `StudentDetailsFormContent`

**`StudentDetailsFormContent.tsx`** (client component — `"use client"`):
- `selectedCourse` state drives AOS filtering
- Area of Study dropdown disabled with "Select a course first" until a course is chosen
- Submitted values are **codes** (e.g. `C2001`, `ALGSFTWR01`), not titles

## algo1.py Interface

```bash
python3 algo1.py --course C2001 --specialisation ALGSFTWR01 --campus Clayton --output schedule_<uuid>.json
```

- Default campus: `Clayton`
- Default output: `schedule.json` (in same dir as script)
- Output format: JSON matching the `Schedule` type above
- Semester labels: `"Year 1, Semester 1"`, `"Year 1, Semester 2"`, etc. (relative years, not calendar)
- Units with code `"ELECTIVE"` are free elective placeholders

**Python command fallback** (in `profile/page.tsx`):
- Windows: tries `py` → `python` → `python3` (detects via `process.platform === "win32"`)
- Unix/Mac: tries `python3` → `python` → `py`
- Skips a command on `ENOENT` (Unix not found) or exit code `9009` (Windows not recognized)

## Authentication & Database

- **Clerk** handles all auth. Server components use `await currentUser()`.
- **Supabase** `profiles` table: one row per user, keyed by `email`. `plans` column is a JSONB array of `Plan` objects.
- Profile auto-created on first dashboard visit.
- `searchParams` in Next.js 15+ is a **Promise** — must `await searchParams` before accessing properties.

## Unit Reviews (Reddit Side Panel)

**`UnitReviewModal.tsx`** (client component — despite the name, it's a right-side drawer panel):
- Opens when any non-ELECTIVE unit card is clicked in `CoursePlanner`
- Fetches `GET /api/units/[code]/reviews` on mount
- Shows loading spinner → list of Reddit posts (title, upvotes, comment count, external link) → empty state if none found
- Close via ✕ button, clicking the backdrop, or pressing Escape

**`/api/units/[code]/reviews/route.ts`**:
- Searches `r/Monash` via Reddit's public JSON API (`reddit.com/r/Monash/search.json`)
- Query: `<code> OR <numeric-part>` (e.g. `FIT2004 OR 2004`)
- Filters results to only posts whose title mentions the unit code or number
- Cached for 1 hour (`revalidate: 3600`)
- Returns: `{ title, url, score, numComments, subreddit }[]`

## Current Limitations

- Only Monash University is supported (hardcoded in form + algo)
- Only "February" semester offering available (July not yet added)
- Category enrichment is simplified: AOS units → Specialisation, ELECTIVE → Elective, everything else → Core (no Major/Minor distinction yet)
