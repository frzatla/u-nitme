# U-NIT ME — CLAUDE.md

## Project Overview

**U-NIT ME** is a Next.js web app that helps university students plan their courses using AI-driven recommendations. Built for a Unihack hackathon. Users sign in, fill in their degree details, and get a semester-by-semester course plan.

## Key Data Models

```typescript
// lib/types.ts
type Plan = {
  id: string;
  planName: string;
  courses: string;      // course_code from final_courses.json (e.g. "C2001")
  university: string;
  areaOfStudy: string;  // AOS code from final_aos.json (e.g. "ALGSFTWR01")
  semesterOffering: string;
  yearStart: number | string;
  yearEnd: number | string;
};

type Profile = {
  id?: string;
  email: string;
  plans: Plan[];
};

type ChosenUnit = {
  chosenId: number;
  unitId: number;
  unitName: string;
  chosenYear: number;
  chosenSem: string;
  type: string;        // "Core" | "Major" | "Minor" | "Elective"
  creditPoint: number;
  difficulty: string;
};
```

**CoursePlanner-internal types** (defined in the component):
- `Unit` — code, name, category, level, credit points
- `Semester` — units + year/title metadata
- `YearGroup` — semesters grouped by academic year
- `Summary` — totals (units, credits, progress %, category breakdown)

## Authentication & Data Flow

1. User authenticates via **Clerk** (handles all sessions/redirects)
2. Server components call `await currentUser()` — redirect to `/sign-in` if unauthenticated
3. On first dashboard visit, a **Supabase profile** is auto-created (keyed by email)
4. Plans stored as a JSON array inside the profile row
5. Form submission uses **Next.js Server Actions** (`"use server"`) → saves plan → redirects to `/course-plan`

**Middleware** (`proxy.ts`): Clerk middleware runs on all routes except Next.js internals and static files. Always runs for `/api/*`.

## Course Plan Generation

- `CoursePlanner.tsx` contains a hardcoded `foundationUnits` array (12 Monash IT foundation units)
- After foundation units, placeholder units are generated by category
- Default: 4 units/semester, 8 semesters (2-year program), target 28 total units
- Progress = `(completedUnits / 28) * 100`
- Category color coding: Core (blue), Major (green), Minor (yellow), Elective (orange)

## Conventions

- **Path alias**: `@/*` maps to `./src/*`
- **Server components** for data fetching; `"use client"` only when interactivity/hooks needed
- **Server Actions** for form submissions (no separate API route needed for forms)
- **Tailwind v4**: uses `@tailwindcss/postcss` plugin (not the old `tailwind.config.js`)
- TypeScript strict mode is **disabled** — avoid adding strict-only constructs
- Framer Motion for all animations — use `motion.div` with `whileInView` / `viewport` for scroll reveals

## Student Details Form

`StudentDetailsForm.tsx` is a server component that:
1. Reads `final_courses.json` via `fs.readFileSync` — builds a sorted list of all courses
2. Reads `final_aos.json` — filters to only entries where `total_credit_points > 0`
3. Builds `courseToAos: Record<string, string[]>` — maps each course code to its valid AOS codes
4. Passes all three as props to `StudentDetailsFormContent`

**AOS extraction rule** (in `StudentDetailsForm.tsx`): a requirement group contributes AOS codes only if:
- `credit_points > 0` — excludes optional "Discipline elective studies" groups (cp: 0)
- **All** units in the group are valid AOS codes — excludes mixed groups with FIT/MAT unit codes

This ensures e.g. C2001 shows only `ATINTELL02, DATASCI11, ALGSFTWR01, CSCYBSEC01` (from "Part C. Specialist studies", cp: 36), not the optional minors from the electives group.

`StudentDetailsFormContent.tsx` is a **client component** (`"use client"`) that:
- Tracks `selectedCourse` in `useState`
- Filters `aosList` to `courseToAos[selectedCourse]` on course change
- Area of Study dropdown is disabled with "Select a course first" until a course is chosen
- Selecting a new course resets the AOS selection

Submitted form values for `courses` and `areaOfStudy` are **codes** (e.g. `C2001`, `ALGSFTWR01`) — matching what `algo1.py` expects via `--course` and `--specialisation` flags.

## Public Data Files

All located in `public/data/`:

| File | Structure | Key field |
|---|---|---|
| `final_courses.json` | Object keyed by `course_code` | `course_title`, `total_credit_points`, `requirement_groups`, `all_units` |
| `final_aos.json` | Object keyed by AOS code | `course_title`, `total_credit_points` — filter `> 0` for valid AOS |
| `final_units.json` | Unit database | Used by `algo1.py` for scheduling |

`algo1.py` loads all three files relative to its own directory (`../../public/data/`).
